Environment Setup and Execution:

1. Install Docker Desktop.

2. Start the application with Docker:
   - Navigate to the project root.
   - Run 'docker compose up --build'.
   - This starts the frontend, backend, and MongoDB together.

3. Access the application:
   - Frontend: http://localhost:3000
   - Backend API: http://localhost:5001
   - MongoDB: mongodb://localhost:27017

4. Notes:
   - The backend container listens on port 5000 internally and is mapped to host port 5001.
   - Do not open MongoDB port 27017 in a browser; use mongosh or MongoDB Compass instead.
   - There is no seed data by default, so the collections may start empty.

5. Optional local backend setup:
   - If you run the backend outside Docker, create a '.env' file in the 'backend' directory with:
     DATABASE_URL=mongodb://127.0.0.1:27017/inventory
     JWT_SECRET=your_secret_key
     PORT=5000

Testing and Issues:

- User authentication may have issues with token expiration.
- Product list rendering might be slow with a large number of products.
- Report generation may fail for certain date ranges.
- Some transaction entries might not update stock levels correctly.
- Responsiveness issues may be present on smaller screens.
- Concurrent transaction handling during high traffic may fail.
- Database connection may fail with high load.
- Some edge cases with forms have not been handled.
- Some pages don't render correctly.
- Some api calls don't return the correct data.
